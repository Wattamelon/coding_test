# 콜라 문제 (일반화) review

## 1) 문제에서 습득해야 하는 핵심 개념
### A. 시뮬레이션(그리디) 방식의 상태 전이
- 현재 빈 병이 `n`개일 때
  - `q = n // a` : 교환 가능한 횟수
  - `r = n % a`  : 교환하고 남는 빈 병
  - 이번에 새로 받는 콜라(=새 빈 병) `gained = q * b`
  - 다음 상태는 `n = gained + r`
  - 누적 정답은 `answer += gained`
- 이 로직을 `n < a`가 될 때까지 반복하면 정답.

### B. (중요) “변수 의미가 고정”되게 코드 짜기
- 코테에서 실수 줄이는 핵심은 “변수 하나가 의미가 계속 바뀌지 않게” 만드는 것.
- `q, r, gained`처럼 역할을 분리하면 디버깅/검증이 쉬워짐.

### C. 수식(수학적 관찰)로 한 방에 푸는 관점
- 교환 1번은 `a`개를 내고 `b`개를 받는 것 → 빈 병 총량은 결과적으로 `a-b`만큼 감소하는 효과.
- 교환이 가능한 “횟수”는 대략 `(n-b)//(a-b)`로 떨어지고,
- 한 번 교환할 때마다 `b`병을 받으므로 최종 콜라 수는 `* b`.

---

## 2) 코테용 베스트 코드 (solution_byGPT.py)
```python
def solution(a, b, n):
    answer = 0
    while n >= a:
        q, r = divmod(n, a)   # q: 교환 횟수, r: 남는 병
        gained = q * b        # 이번에 새로 받은 콜라 수
        answer += gained
        n = gained + r        # 새 빈 병 + 남는 병
    return answer
