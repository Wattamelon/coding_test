# 3진법 뒤집기 review

## 1) 문제 핵심 요약
- 해야 할 일은 3단계다.
  1) n을 3진법으로 변환
  2) 3진법 문자열(또는 자리)을 앞뒤로 뒤집기
  3) 뒤집힌 3진법을 다시 10진법으로 변환

- 핵심 포인트는 “3진법 변환”을 직접 구현할 수 있어야 한다는 것.

---

## 2) 반드시 알아야 하는 기술(코테용)

### A. 10진수 → k진수 변환 원리 (가장 중요)
- 어떤 수 n을 k로 계속 나누면 나머지들이 k진수의 자릿수가 된다.
- 규칙:
  - `n % k`  : 현재(가장 낮은) 자릿값
  - `n // k` : 다음 단계로 넘어갈 몫
- 나머지를 “먼저 나온 것부터” 모으면 그건 k진수의 **역순(뒤집힌)** 형태가 된다.

예시) n=45를 3진법으로
- 45 ÷ 3 = 15 ... 0
- 15 ÷ 3 = 5  ... 0
- 5  ÷ 3 = 1  ... 2
- 1  ÷ 3 = 0  ... 1
나머지(아래자리부터): 0,0,2,1
정상 3진수: 1200(위에서부터)
뒤집힌 3진수: 0021(아래에서부터 그대로 쓰면 뒤집힌 꼴)

즉,
- “나머지를 append한 리스트” = 이미 “뒤집힌 3진수”

---

### B. 뒤집힌 3진수 → 10진수 변환
- 뒤집힌 3진수 문자열이 있으면:
  - `int(base3_string, 3)` 로 바로 10진수 변환 가능 (파이썬 기능)

- 문자열 없이 직접 계산도 가능:
  - 뒤집힌 3진수의 자릿값 r을 순서대로 읽으면서
  - `res = res * 3 + r` 로 누적하면 된다.
  - (k진수 숫자를 왼쪽부터 읽으며 10진수로 바꾸는 표준 방식)

## 3) 코테용 베스트 풀이 아이디어 (solution_byGPT.py)
- 3진수 변환을 할 때, `r = n % 3`는 항상 “맨 끝 자리”가 나오는데
- 우리가 필요한 건 “3진수를 뒤집은 결과”이므로
- 그 나머지들을 나온 순서대로 바로 3진수로 누적하면 끝난다.

## 4) 내 약점(3진수 자체를 떠올리지 못함) 보완법
Step 1) “진법 변환 = 나누기 + 나머지” 패턴을 그냥 외워도 됨

코테에서 진법 변환은 거의 항상 이 패턴이다.

공식처럼 기억:

while n > 0: 
    n, r = divmod(n, k); 
    digits.append(r)

Step 2) 문제에서 “진법” 키워드가 나오면 체크리스트 돌리기

아래 단어가 보이면 바로 진법 변환을 의심한다:

“k진법”, “n진법”, “2진수/3진수”, “뒤집기”, “변환”, “표현”

특히 “뒤집기”가 붙으면:

나머지 append 순서가 이미 뒤집힌 형태라는 걸 떠올리면 쉬워진다.

Step 3) 10분 훈련(추천)

아래를 손으로/코드로 3~5번만 해도 패턴이 몸에 붙는다.

10진수 → 2진수, 3진수, 8진수로 바꾸기

바꾼 결과를 다시 10진수로 되돌리기(res = res*k + digit)

# 10진수 n -> "정상" 3진수 문자열(뒤집지 않은 형태)
def to_base3(n: int) -> str:
    digits = []
    while n > 0:
        n, r = divmod(n, 3)
        digits.append(str(r))   # r은 3진수의 "맨 뒤 자리"부터 나온다
    return "".join(reversed(digits))  # 뒤집어서 정상 순서로

# 예시
print(to_base3(45))  # 1200

# 참고: "뒤집힌 3진수" 문자열은 그냥 뒤집지 않고 그대로 join 하면 됨
def to_reversed_base3(n: int) -> str:
    digits = []
    while n > 0:
        n, r = divmod(n, 3)
        digits.append(str(r))
    return "".join(digits)  # 이미 뒤집힌 상태

# 예시
print(to_reversed_base3(45))  # 0021
