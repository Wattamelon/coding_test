[왜 이 풀이가 정석인가]
- 사이트 주소 -> 비밀번호를 빠르게 찾아야 함
- N, M이 각각 100,000이라 선형탐색(list)로 찾으면 O(N*M)이라 터짐
- dict(해시맵)을 쓰면
    - 저장: O(N)
    - 조회: O(M)
  평균적으로 각각 O(1) 조회라 전체 O(N+M)

[네 코드 장점]
- input = sys.stdin.readline 사용 ✅
- dict로 매핑 저장 ✅
- 출력도 join + stdout.write로 한번에 ✅
- order를 리스트로 모아둔 뒤 한번에 출력하는 것도 괜찮음(메모리도 충분)

[아주 미세하게 더 “코테스럽게” 개선할 수 있는 점(선택)]
- order 리스트를 굳이 만들지 않고, 읽으면서 바로 out에 append 해도 됨
  (하지만 지금도 충분히 빠르고, 메모리 10만 줄 정도는 문제 없음)

[결론]
- 이 문제에서는 네 코드가 거의 최적해(정답+성능+가독성 다 잡음)
